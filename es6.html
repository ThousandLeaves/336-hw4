About ECMA Script 6

ECMA Script 6 (Known also as ES6) is a major revision for the Javascript standard that was released in 2015. While there have been more revisions since that time, ES6 is notable for a number of new functionalities that was designed to simplify some functions of Javascript while simultaneously modernizing the language. Because ES6 was released in 2015, that means its features are safe to use on all modern browsers. There is little reason not to use them!

This page will only go over several of the most significant updates, features that any Javascript developer should be using.

The Let Statement

In Javascript, variables should now be declared by using the let statement.

let myVariable = "Welcome to the Javascript Notebook."

The let statement works by giving block scope to all variables declared in this way. Block scope means that any variable placed within brackets { } will only be visible inside those brackets. This means a variable declared using let in an if statement will only be visible in that statement.

On the Internet, there are still many websites and tutorials that instruct new users of Javascript to get comfy with declaring their variables by the var statement. This does not work the same way as let. Variables declared with var possess function scope instead, meaning that all variables declared in a function are visible anywhere within the function. Since ES6 though, the Let statement should be considered the standard use case for declaring most variables. While var may not be deprecated, there are no real advantages to using it.


The Arrow Function Expression

ES6 provides an alternative way to create function expressions. The main advantage to arrow functions is that they are easier to read and contain an implicit return (Unless you use the brackets style). This means you must return some value using them though.

let myArrow = (x, y) => Math.pow(x, y);
console.log(myArrow(2,3))

let myArrow2 = (x, y) => { return Math.pow(x, y) };

Expected outputs for both: 8


In the two examples above, we are declaring a variable and setting it equal to a function, a legal move in Javascript known as a function expression. Both of these examples use the arrow expression form, but pay close attention to the second use case where brackets are included: Brackets allow for the arrow expression to occupy multiple lines, but this form requires an explicit return statement.


Default Parameter Values

Prior to ES6, when creating a function definition it was not possible to define default values for the function parameters. Of course this means that using a function without declaring a specific value will result in the default one rather than undefined.

function addition(a, b = 4) {
return a + b;
}
console.log(addition(2))

Expected result: 6

In this brief example, its evident that the function would normally require two arguments. Since a default value is specified for the parameter b, the second value falls back to the default of 4.
